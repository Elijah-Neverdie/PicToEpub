# 📖 Pic2EPUB 使用说明

**Pic2EPUB** 是一个将图片文件夹一键转换为标准 EPUB 电子书的小工具。支持多语言界面（中文 / English）、批量处理、自动封面识别、智能排序，并提供覆盖策略控制。

## ✅ 功能特性

- 🖼️ 支持常见图片格式：JPG、PNG、GIF、WebP、SVG 等（依赖 Pillow 库）
- 📘 自动生成带目录、元数据和分页的 EPUB 3.0 格式电子书
- 🔍 自动识别含 `cover` 的文件作为封面（优先 JPG）
- 🔢 图片按文件名中的数字智能排序（如 `01.jpg`, `page2.png`）
- 📁 批量处理：
  - **单独模式**：每个子文件夹生成独立 EPUB
  - **合并模式**：所有子文件夹图片合并为一本 EPUB
- ⚠️ 智能覆盖提示：跳过 / 覆盖 / 取消 + “应用于所有”选项
- 🌐 多语言界面：中文 / English（可扩展）
- 🧹 自动清理临时文件，无需手动干预

## 🛠️ 安装与依赖

### 系统要求
- Python 3.6 或更高版本
- Windows / macOS / Linux 均可运行（GUI 基于 Tkinter）

### 首次运行
首次启动时，程序会自动检测并提示安装所需依赖：
- `Pillow`（用于图像处理）
- `lxml`（用于 XML 构建）

> 若网络受限，请手动安装：
```bash
pip install Pillow lxml

使用步骤
运行程序
Bash
编辑
python pic2epub.py
选择图片文件夹
点击【浏览...】按钮，选择包含图片的文件夹。
文件夹中不能只包含 .epub 文件（会被自动忽略）。
处理方式选择（如有子文件夹）

若检测到 ≥1 个含图片的子文件夹，将弹出选项：
✅ 为每个子文件夹单独生成 EPUB
🔗 将所有图片合并到一个 EPUB
❌ 取消操作
覆盖策略（如输出文件已存在）

弹出对话框询问如何处理同名文件：
跳过：不生成该 EPUB
覆盖：替换原文件
应用于所有：对后续所有冲突采用相同操作
等待生成完成
进度条显示扫描、处理、打包各阶段状态
支持随时点击【取消】中断操作

单独模式：文件夹名.epub

失败：弹出错误信息

📁 文件结构示例
单文件夹模式
Text
编辑
漫画/
├── 01.jpg
├── 02.jpg
├── cover.jpg   ← 自动识别为封面
└── (生成) 漫画.epub

批量子文件夹模式
Text
编辑
系列漫画/
├── 第1话/
│   ├── 01.jpg
│   └── ...
├── 第2话/
│   ├── page1.png
│   └── ...
└── (生成) 第1话.epub, 第2话.epub   ← 单独模式
或

⚠️ 注意事项
图片文件名建议包含数字（如 001.jpg），否则排序可能不符合预期。
封面文件需包含 cover 字样（不区分大小写），如 Cover.jpg、my_cover.png。
输出 EPUB 默认语言设为 zh（中文），可修改源码调整。
不支持嵌套多层子文件夹递归扫描（仅扫描一级子文件夹）。
